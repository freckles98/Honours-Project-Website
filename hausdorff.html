Hausdorff Distance

Project Goals

In this approach, we use the Hausdorff Distance paired with and optimisation algorithm to identify planting patterns within trees. We use the centroid of each tree to represent the tree and identify patterns within the points. The Hausdorff Distance assesses the degree of similarity between a planting pattern and the data. Three different Hausdorff methods are used to evaluate the data and determine the best approach. The optimisation algorithm finds the best matches over the entire data set. The aim of the project is:


1. Evaluate the which Hausodrff method is best suited to the problem.
2. Determine the robustness and feasibility of the approach for determining planting patterns.

Algorithm

To implement the algorithm the following steps were taken:


Step 1
Creating model sets of of the patterns needing to be indentified. There were square, rectangle, quincunx, double row and heaxagonal.

Step 2
Using the polygon centroids as point and translating and rotating this on to an effective coordinate reference system.

Step 3
Determining the density of the data and scaling effectively to ensure 1 by 1 unit average between trees.

Step 4
Choosing the Hausdorff distance to be tested. There were three options: Modified Exact, Partial and Average Hausdorff Distance.

Step 5
Running the matching algorithm that performs rotations and translations of the model to effectively match it to the data.

Step 6
Repeating step 5 through the data set to find the pattern with the lowest Hausdorff Distance and thus the identified pattern.


Hausdorff Distance Variations

There are three variations of the Hausdorff Distance found in literature. These methods are:

1. Modified Exact
2. Partial
3. Average

They were all implemented within the experiement to asses viability. As the search for the nearest neighbour is a computaionally expensive calculation of ğ‘‚(ğ‘›^2), a CKD-tree is used. The modified exact Hausdorff distance proved to be too strict on inconsistencies between the data and thus could not determine the pattern accurately.
The partial Hausdorff distance was too lenient when identifiying the patterna and identified multiple patterns within one portion of the image.
The average Hausdorff distance proved to be the best metric as it was lenient with noisy data, but still managed to effectively identify the pattern.

Matching Algorithm

When matching the model to the data it is unlikely that they will line up correctly, thus we implement a matching algorithm to effectively find the correct matches and thus reduce the Hausdorff distance. The model is translated through a small portion of the data in 1 by 1 units for the square, hexagonal and quincunx shape and 1 by 2 units for the rectangle and double row shape. This reduces computational complexity as the uniform allows for this.
The next issue to consider is rotations. The data not necessarily going to be aligned in the correct formation. Thus each shape needs to be rotated within each increment, thus adding additional computational complexity. In attempt to reduce this, a least squares regression line is used to correctly align the data. However, this did not work correctly and future works may want to look at the Hough transform to achieve this.

Findings


The approach could correctly identify the rectangular pattern in all given data sets. However, where the Hausdorff distance window was too small, incorrect patterns were detected, i.e. on the border.

Detecting a round or boundary is not possibly using the Hausdorff Distance approach. It is a very challenging problem to solve and after 30 years of research there is still no road detection published or on the market. Thus it was not achieved in this paper.

The average Hausdorff distance was the best at aligning data and correctly identifying the planting pattern. The table on the right shows all the tests run on datasets and allocates a number 1 to 5 to each test. 5 being the best score and 1 being the worst score. This clearly demonstrates how the average Hausdorff outperforms the other approaches. These tests also demonstrate the length of time taken for each experiment. It is clear that the algorithm is not computationally efficient and very inefficient when implementing rotations.

Conclusions

The Hausdorff distance performs well when aggregating points within noisy datasets and is able to detect the planting pattern within the provided data. However, as most of the data supplied contains rectangular patterns, it cannot be pronounced a robust method until tested on more data containing the patterns to be detected.

The matching algorithm needs to account more accurately for different window sizes by implementing a better approach that accounts for more trees within the window. Reducing runtime is the most important aspect of improving this implementation. This can be achieved through finding an effective pruning process.
