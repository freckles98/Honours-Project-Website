Hough Transform

Project Aims

Primary Aim:
To detect planting patterns from aerial footage of orchards using the Hough Transform.


Secondary Aims:
Extract key parameters about the orchard such as orientation, intra-row, inter-row spacing and boundary points.

What is the Hough Transform?

The Hough Transform is an image processing technique that can be ued for the detection of lines in digital images. One of the main features of this process, is its ability to detect a line, regardless of how fragmented it is.

The HT method follows three main steps:

Firstly, the coordinates of a pixel in the cartesian image space is transformed into a sinusoidal curve in the parameter space. Secondly, an accumulator with a cell array (which is equal to the size of the unknown dimensions) is laid on the parameter space and each pixel adds one vote to the cells lying on its sinusoidal curve. Thirdly, cells with global or local maximum votes are selected and the parameter coordinates (rho, theta) can be used to determine the line as well as its orientation in the cartesian space

This technique is applied in order to identify the primary and secondary aims.

Pre-Processing




Before the Hough Transform can be applied, input data is pre-processed which was done in the following steps:


Step 1: Finding Tree Centroids

Centroids are found from the GeoJson data containing polygon information of tree locations at different confidence levels. This information is plotted and saved as an image file.


Step 2: Edge Detectioin

The Hough Transform takes an edge map of the plotted data as input. In order to achieve this, a Canny Edge Detector is used, which identifies the edges of the centroids within the image.


Step 3: Pixel Reduction

Application of the Canny Edge detector results in an edge map where each tree centroid is represented by multiple pixels. To increase accuracy of the Hough Transform, these pixel centroids are reduced to single pixels.

Hough Transform Application

After the data is pre-processed, the HT can be applied and the results from this process are then used in the post-processing step to extract the main and secondary directions needed to identify the planting pattern. as well as secondary aims.

Step 1: HT Application to find Orientation

The Hough transform is used at increasing threshold values in order to obtain the main orientation of the orchard structure, represented by the theta value. This is further used to extract the secondary orientation of the orchard which is at 90 degrees to the main orientation.


Step 2: HT applied at low and high thresholds

The theta values for the main and secondary orientation are used to extract the rho values which represent the positions of the rows within the orchard structure for each direction.

The application of the HT results in multiple rho values being determined for each of the main and secondary direction. These values need to be post-processed in order to reduce false positive rows. That output can then be used identify the correct rows within the orchard structure, leading to pattern detection and extracting secondary aims.

Post-Processing

Output from the HT contains excess information that needs to be reduced so only the correct information about the rows in the orchard structure is kept.

Step 1: Line Filtering

Due to centroid variations within rows leading to multiple lines representing the same row in an orchard, line filtering is used to combine these into single orchard rows. This is based on a margin of error between HT output rho values. false positive row identification.


Step 2: Conversion from Hough Space to Cartisian Space

Filtered rows represented in Hough space as Rho and Theta values are converted into cartisian (x,y) coordinates so distance measures can be applied in order to classify planting patterns and identify secondary parameters.


Step 3: Pattern Classification and Parameter Extraction

Distances between rows are calculated using a Euclidean distance measure. This is done in both the main and secondary orientation in order to extract inter and intra row spacing. These distances are analysed in order to classify each direction into the correct planting pattern.


Step 4: Intercepts used for corner point identification

The intercept points of the outside rows of both directions are used in order to identify the corner points of the orchard structure. The surrounding area around these points are searched to match pixels in that location.

Results

Primary and secondary aims were attainable from idealized data. However, when applied to Real data sets the variation of centroid location information, especially for the secondary direction led to orchard rows not being identified correctly. This had the result of poor pattern classification due to incorrect rows being located.

Application of a HT threshold value that was too high led to rows represented by fewer points not being identified. When the HT threshold is too high, single rows are identified multiple times even after line filtering is applied.

Ground noise at lower confidence levels led to excess rows being identified, resulting in poor pattern classification.

Missing corner centroids at higher confidence levels led to difficulties in identifying corner positions as intercepts of outside rows cannot be matched to pixels at these positions.

Sporadic placing of centroids leading to False Positive row identification
Ground Noise resulting in False Positive lines being identified

Conclusions

The Hough Transform was successful in identifying primary and secondary aims for idealized data. It did however fall short when applied to real data sets, mainly due to centroid alignment variation of either the main or secondary planting direction.


Selection of the correct HT threshold value was dependent on the dataset. If this was too high, then not all the rows were detected. If this was too low, multiple false positive lines were identified.


The application of the HT was good at identifying the orchard orientation and could be used as a pre-processing step in Template Matching techniques.
